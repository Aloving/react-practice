# Задача 02 — Дебаунс поиска и отмена запросов

**Уровень:** средний

## Цель

Поле поиска с debounce и отменой предыдущего fetch при новом вводе (AbortController). Показать скелетон/лоадер и корректно обработать race condition (если быстрый ввод дал два запроса, в UI должен отобразиться результат последнего).

## Требования

- Input с поиском; запрос на API (или мок) отправляется с задержкой (debounce, например 300–400 ms).
- При новом вводе предыдущий fetch отменяется через `AbortController`.
- Во время загрузки показывать скелетон или индикатор загрузки.
- Если пришёл ответ от более старого запроса, чем текущий ожидаемый — игнорировать его (по счётчику или по AbortController).
- Обработать ошибку отмены (abort) так, чтобы не показывать её пользователю как ошибку.

## Подсказки

- `useRef` для AbortController и для идентификатора последнего запроса.
- Debounce: `useEffect` с таймером и cleanup при размонтировании и при смене query.
- В fetch передать `signal: controller.signal`; перед новым запросом вызвать `controller.abort()` и создать новый контроллер.

## Критерии приёмки

- Запрос уходит только после паузы в вводе (debounce).
- При быстром вводе старые запросы отменяются.
- В UI отображается результат только последнего запроса; нет «мелькания» старых результатов.
- Тест: при смене запроса до ответа отображается результат последнего.
