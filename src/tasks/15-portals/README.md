# Задача 15 — Portals: модалка и оверлей

**Уровень:** продвинутый

## Цель

Реализовать модальное окно через `createPortal`: рендер вне родительского DOM (в `document.body`), оверлей, закрытие по Escape и по клику вне модалки. Обеспечить focus trap для доступности (фокус не уходит за пределы модалки при Tab).

## Требования

- Компонент `Modal`: принимает `open`, `onClose`, `children`.
- При `open === true` рендерить контент через `createPortal` в `document.body` (или в отдельный контейнер-портал).
- Оверлей (полупрозрачный фон) под контентом модалки; клик по оверлею вызывает `onClose`.
- Клавиша Escape вызывает `onClose`.
- Focus trap: при открытой модалке фокус остаётся внутри (Tab циклически перемещается между фокусируемыми элементами модалки; не уходит на элементы «под» оверлеем).
- При закрытии — вернуть фокус элементу, который его открыл (если применимо).
- Опционально: блокировка скролла body при открытой модалке.

## Подсказки

- `ReactDOM.createPortal(children, document.body)` — второй аргумент — контейнер (DOM-узел).
- `useEffect` для подписки на `keydown` (Escape); cleanup при размонтировании.
- Focus trap: `querySelectorAll` по фокусируемым элементам внутри модалки; при Tab на последнем — переводить на первый; при Shift+Tab на первом — на последний.
- `aria-modal="true"`, `role="dialog"`, `aria-labelledby` для доступности.

## Критерии приёмки

- Модалка рендерится вне родителя (проверить в DevTools: контент в body).
- Escape и клик по оверлею закрывают модалку.
- Фокус задерживается внутри модалки при навигации Tab.
- После закрытия фокус возвращается на триггер (если есть кнопка открытия).
