# Задача 08 — Контролируемая форма с валидацией

**Уровень:** средний

## Цель

Форма с несколькими полями, локальная валидация, отображение ошибок, кнопка отправки disabled при невалидных данных.

## Требования

- Несколько полей (например: email, пароль, подтверждение пароля, имя).
- Контролируемые инпуты (value из state, onChange обновляет state).
- Валидация: формат email, минимальная длина пароля, совпадение паролей и т.п.
- Ошибки показывать под полями или рядом; не блокировать ввод.
- Кнопка «Отправить» disabled, пока форма невалидна; при валидной форме — включена (отправка может быть заглушкой: alert или console.log).

## Подсказки

- Один state-объект для всех полей или отдельный state на поле.
- Функция валидации возвращает объект с ошибками по полям; при пустом объекте форма валидна.
- Можно валидировать на blur или при каждом изменении — на твоё усмотрение.

## Критерии приёмки

- Невалидные данные отображаются как ошибки.
- Кнопка отправки неактивна при наличии ошибок.
- После исправления ошибок форма становится валидной и кнопка активна.
