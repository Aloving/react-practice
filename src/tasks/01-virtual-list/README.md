# Задача 01 — Виртуальный список

**Уровень:** средний

## Цель

Реализовать виртуализированный список: рендерить только видимые элементы плюс небольшой буфер сверху/снизу, остальное — placeholder по высоте (чтобы скролл и общая высота контейнера сохранялись).

## Требования

- Список принимает массив элементов и функцию рендера одного элемента.
- Высота контейнера и высота одного элемента (или функция для динамической высоты) задаются пропсами.
- При скролле пересчитывать видимый диапазон индексов и рендерить только их + буфер (например, 5 элементов сверху и снизу).
- Остальное пространство заполнять пустыми div'ами с заданной высотой, чтобы scrollHeight и поведение скролла были корректными.
- Избегать лишних ре-рендеров: не пересчитывать видимый диапазон на каждый requestAnimationFrame, использовать разумный throttle или Intersection Observer при необходимости.

## Подсказки

- `useRef` для контейнера скролла и для хранения размера/позиции.
- Вычисление `startIndex` и `endIndex` по `scrollTop`, `clientHeight` и высоте элемента.
- Список фиксированной высоты элемента проще; переменная высота — продвинутый вариант (можно завести массив высот или оценивать по умолчанию).

## Критерии приёмки

- Рендер только видимых + буфер элементов.
- Плавный скролл без лагов при списке из 10 000+ элементов.
- Тест: рендер списка из 10 000 элементов не создаёт 10 000 DOM-узлов.
