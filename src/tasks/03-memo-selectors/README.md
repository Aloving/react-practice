# Задача 03 — useMemo, селекторы и тяжёлые вычисления

**Уровень:** средний

## Цель

Список с фильтрами и сортировкой. Вынести фильтрацию и сортировку в `useMemo`, чтобы тяжёлые вычисления не перезапускались без необходимости (зависимости — только исходный список, фильтры и порядок сортировки).

## Требования

- Список элементов (например, товары с полями: id, name, price, category).
- Фильтры: по категории, по диапазону цены, по строке поиска в названии.
- Сортировка: по имени, по цене (возрастание/убывание).
- Вычисление отфильтрованного и отсортированного списка — в `useMemo` с правильными зависимостями.
- Имитировать «тяжёлую» операцию (например, искусственная задержка или цикл), чтобы было видно, что без useMemo при каждом ре-рендере список пересчитывается, а с useMemo — только при смене данных/фильтров.
- Не вызывать тяжёлые вычисления в рендере вне useMemo.

## Подсказки

- Зависимости useMemo: `[items, categoryFilter, priceMin, priceMax, searchQuery, sortBy, sortOrder]`.
- Для демонстрации «тяжести» можно добавить в функцию `for (let i = 0; i < 1e6; i++) {}` или `performance.now()` и логировать время.

## Критерии приёмки

- Фильтрация и сортировка работают корректно.
- При изменении несвязанного state (например, счётчик в том же компоненте) отфильтрованный список не пересчитывается.
- Тест: мок тяжёлой функции вызывается только при смене зависимостей.
