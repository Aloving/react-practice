# Задача 19 — WebSocket, reconnect и оффлайн‑очередь

**Уровень:** продвинутый

## Цель

Собрать реального времени список сообщений (чат/лента), который умеет переподключаться и корректно отправлять сообщения после восстановления связи.

## Требования

- Соединение:
  - подключение к WebSocket при монтировании;
  - индикатор статуса: `connecting / online / offline`.
- Приём сообщений:
  - входящие сообщения добавляются в список в реальном времени.
- Отправка сообщений:
  - форма ввода + кнопка отправки;
  - если соединения нет — сообщение попадает в очередь (помечено как “pending”).
- Переподключение:
  - автоматический reconnect с экспоненциальной задержкой (например, 500 мс → 1 с → 2 с → 4 с, max 10 с);
  - после reconnect все сообщения из очереди отправляются по порядку.
- Очистка:
  - корректно закрыть сокет и таймеры при размонтировании.

## Подсказки

- Если реального сервера нет, сделай “фейковый” WebSocket‑слой: имитируй входящие сообщения через `setInterval` и случайные разрывы соединения.
- Для очереди удобно хранить массив “pending” сообщений с локальным `id`.

## Критерии приёмки

- При потере соединения UI остаётся рабочим, отправка складывается в очередь.
- После восстановления связи очередь отправляется в правильном порядке.
- Индикатор статуса корректно отражает текущую фазу подключения.
