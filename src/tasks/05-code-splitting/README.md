# Задача 05 — Code splitting и ленивая загрузка маршрутов

**Уровень:** продвинутый

## Цель

Использовать `React.lazy` и `Suspense` для маршрутов страниц. Реализовать прелоад следующего маршрута при hover на ссылку (или при появлении в viewport). Измерить уменьшение начального бандла (до/после).

## Требования

- Несколько «страниц» (компонентов): Главная, О проекте, Дashboard, Настройки. Каждая — отдельный чанк (lazy).
- Маршрутизация: можно без react-router — свой простой роутер по `window.location.hash` или по state (например, `page === 'home' | 'about' | 'dashboard' | 'settings'`).
- При переходе по ссылке показывать fallback Suspense (спиннер или скелетон), пока чанк грузится.
- Прелоад: при наведении на ссылку (или при появлении ссылки в viewport) начать загрузку соответствующего чанка, чтобы при клике переход был без задержки или с минимальной.
- В README или в консоли описать: размер начального бандла до и после code splitting (можно через `vite build` и посмотреть размеры чанков в выводе).

## Подсказки

- `React.lazy(() => import('./pages/Dashboard'))` для каждой страницы.
- Прелоад: сохранить результат `import()` в ref или в состоянии «предзагруженных» модулей; при hover вызывать `import('./pages/Dashboard')` без рендера, чтобы при клике модуль уже был в кэше.
- Размеры: `vite build` выводит размеры файлов; до splitting один main chunk, после — main + отдельные chunk для каждой страницы.

## Критерии приёмки

- Переход по маршруту загружает чанк и показывает fallback при необходимости.
- Hover (или viewport) на ссылку запускает прелоад; при последующем клике задержки нет или она минимальна.
- В проекте есть описание/скрин или вывод размера бандла до/после.
