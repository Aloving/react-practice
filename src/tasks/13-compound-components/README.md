# Задача 13 — Compound components и контекст

**Уровень:** senior

## Цель

Реализовать составной компонент (например, Tabs или Accordion): подкомпоненты связаны через контекст, гибкий API разметки, разделение ответственности без prop drilling.

## Требования

- Один составной компонент на выбор: Tabs (вкладки с контентом) или Accordion (раскрывающиеся секции).
- API в виде составных частей: например, `<Tabs><TabList>...</TabList><TabPanels>...</TabPanels></Tabs>`, внутри TabList — несколько `<Tab>`, внутри TabPanels — несколько `<TabPanel>`. Или Accordion: `<Accordion><AccordionItem>...</AccordionItem></Accordion>`.
- Состояние (какая вкладка активна / какая секция открыта) хранится в контексте, предоставляемом корневым компонентом. Дочерние компоненты (Tab, TabPanel и т.д.) используют контекст для координации.
- Гибкая разметка: порядок и состав подкомпонентов можно менять в разметке; стилизация и доступность (aria-атрибуты) приветствуются.

## Подсказки

- React.createContext для значения (активный индекс) и сеттера (setActiveIndex).
- Корневой компонент держит useState и передаёт в Provider.
- Tab при клике вызывает setActiveIndex; TabPanel рендерит контент только если его индекс совпадает с активным.
- Для Accordion: несколько открытых или один — на твоё усмотрение (можно массив открытых id).

## Критерии приёмки

- Составной компонент собирается из подкомпонентов через разметку (compound API).
- Нет prop drilling: состояние и логика в контексте.
- Переключение вкладок / раскрытие секций работает и синхронизировано между частями.
